<objective>
Build `io_mod` (`src/io.f90`): the data-input layer that loads (a) the editable HCMC
district table from CSV and (b) the editable model coefficients from a Fortran namelist,
both with fail-loud per-field validation that returns `stat`/`msg` and **never** aborts.
This is the runtime-editability layer: it delivers GRID-01 (grid loads from an editable
data file, no recompile) and GRID-04 (coefficients editable without recompile), both per
D-01/D-02. It is also the ASVS V5 input-validation surface for the phase — untrusted file
content crosses into the program here, so D-07 fail-loud validation is the security control.

Purpose: Phase 2 physics consumes `grid_t` + `coeffs_t`; nothing downstream can run until
they can be loaded and trusted. Returning `stat`/`msg` (Pattern 4) instead of `error stop`
keeps both loaders unit-testable, which is why a test-drive suite ships in the same plan.

Output: `src/io.f90` with `read_grid_csv`, `read_coeffs_nml`, and the `split_commas` helper;
a `test/test_io.f90` test-drive suite; and `test/fixtures/*` (one valid CSV, four malformed
CSVs, a full namelist, a partial namelist). This plan does NOT ship the real `data/` seed
file and does NOT wire the driver — that is 01-03.
</objective>
