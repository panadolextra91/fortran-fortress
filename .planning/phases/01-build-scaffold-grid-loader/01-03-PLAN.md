<objective>
Ship the HCMC seed data and the real driver that makes Phase 1's goal observable: a runnable
`fpm` program that loads the editable district CSV and the coefficient namelist, validates them
fail-loud, and prints the loaded grid to the terminal. This replaces the 01-01 banner stub with
the actual load -> validate -> print wiring, and adds an end-to-end test proving the shipped
files load with the expected count and that a corrupted copy is rejected with a line number.

Purpose: This is the plan that closes the phase. With 01-01's foundation/types and 01-02's
io_mod loaders in place, the driver composes them into the file->terminal round-trip that
satisfies GRID-01 (editable grid loads + echoes, no recompile) and GRID-03 (realistic seed
archetypes; malformed rows rejected loudly with their line number). It also ships the GRID-04
coefficient data file so the weights/multipliers load at runtime.

Output: `data/hcmc_districts.csv` (~14 districts, D-05), `data/coeffs.nml` (&coeffs namelist),
a real `app/main.f90` driver (D-07 fatal-exit policy + grid print), and `test/test_e2e_load.f90`.
</objective>

    everything from the data files. Do NOT call `error stop` for anything except a loader `stat /= 0`.

| T-01-SC | Tampering | npm/pip/cargo installs | accept | No package installs occur in this plan — it ships data + driver + a test and consumes already-built modules. fpm + test-drive were installed and legitimacy-assessed in 01-01 / RESEARCH "Package Legitimacy Audit" (both official fortran-lang org, verdict OK).
