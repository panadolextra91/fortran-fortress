  <action>
    Create `src/heat_index.f90` defining `heat_index_mod`: `use kinds_mod, only: wp`,
    `use constants_mod, only: ROTH_C1, … ROTH_C9` (the named coefficients from Task 1),
    `implicit none`, `private`, `public :: heat_index_f`. Implement one `elemental pure function
    heat_index_f(t_f, rh) result(hi)` with `real(wp), intent(in) :: t_f, rh` and `real(wp) :: hi`,
    following the verified 4-step NWS order (D-07): (1) simple Steadman
    `hi = 0.5_wp*(t_f + 61.0_wp + (t_f-68.0_wp)*1.2_wp + rh*0.094_wp)`; (2) average with T,
    `hi = (hi + t_f)/2.0_wp`; (3) if `hi >= 80.0_wp`, replace `hi` with the full Rothfusz
    regression built from `ROTH_C1..ROTH_C9` times the T/RH product terms, then conditionally
    apply the two adjustments — subtract `((13.0_wp-rh)/4.0_wp)*sqrt((17.0_wp-abs(t_f-95.0_wp))/17.0_wp)`
    when `rh < 13.0_wp .and. t_f >= 80.0_wp .and. t_f <= 112.0_wp`, and add
    `((rh-85.0_wp)/10.0_wp)*((87.0_wp-t_f)/5.0_wp)` when `rh > 85.0_wp .and. t_f >= 80.0_wp .and.
    t_f <= 87.0_wp`; (4) otherwise keep the step-2 Steadman value. Branch on the step-2 averaged
    value, NOT on raw inputs (PITFALLS B1). Every literal `_wp`-suffixed (D-08, A1). The kernel
    is side-effect-free — no I/O, no stop statement, no validation inside (required by `elemental
    pure`; validation stays in the loader). Do NOT apply any post-hoc offset here — this kernel is
    pure heat index over a given (t_f, rh).
  </action>
