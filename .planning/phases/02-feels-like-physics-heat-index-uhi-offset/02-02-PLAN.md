<objective>
Deliver the additive UHI temperature offset (UHI-01) and its one new tunable coefficient, `d0`.
Add `uhi_mod` — an `elemental pure` kernel computing the single documented budget
`ΔT = w_build·B + w_urban·U − w_tree·V − w_water·Wprox` applied once (D-03), with continuous
water-proximity decay `Wprox = exp(−water_km/d0)` (D-04). Plumb `d0` end-to-end: a new `coeffs_t`
field (default 2.5_wp), loaded from `data/coeffs.nml` via `read_coeffs_nml` (D-05), and guarded
fail-loud (`d0 > 0`) so a bad config cannot inject Inf/NaN into the division. Lock the budget's
sign/monotonicity and single-digit-°C magnitude with a test-drive suite (D-11), and lock the d0
guard with an IO test.

Purpose: this is the spatial-variation engine — every cell's heat differs from baseline only
through this offset (D-01), so its signs and magnitude are what make the urban>rural ordering
true in 02-03.

Output: modified `src/grid.f90`, `src/io.f90`, `data/coeffs.nml`, `test/test_io.f90`; new
`src/uhi.f90`, `test/test_uhi.f90`, `test/fixtures/coeffs_bad_d0.nml`.
</objective>
