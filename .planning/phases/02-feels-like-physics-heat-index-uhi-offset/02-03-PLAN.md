<objective>
Compose and wire the per-cell feels-like temperature and lock the headline ordering invariant.
Add `feels_mod` — a thin `elemental pure` wrapper `feels_like_c` that composes `uhi_offset`
(02-02) and `heat_index_f` (02-01) with the °C↔°F helpers and the D-09 floor, putting the entire
phase formula `feels = max(HeatIndex(t_base + ΔT_UHI, rh), t_adj)` in one tested place (D-01, D-03,
D-08, D-09). Wire it into the existing `app/main.f90` District List loop so each occupied cell
prints `FEELS=` (computed from uniform `coeffs%t_base` + per-cell `cell%rh`). Then lock UHI-02 with
a test-drive suite using synthetic controlled archetypes built inside the test (D-10), plus
property-monotonicity and floor-edge checks (D-11, D-09).

Purpose: this is where HEAT-01 (per-cell feels-like for every cell) and UHI-02 (the make-or-break
dense-urban > green/waterfront/rural ordering) actually become true and CI-locked. The wrapper is
the single named function Phases 3/4 will reuse per timestep / per CSV row.

Output: new `src/feels.f90`, `test/test_ordering.f90`; modified `app/main.f90`.
</objective>
