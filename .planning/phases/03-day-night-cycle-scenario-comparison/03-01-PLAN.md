<objective>
Build the Phase-3 diurnal foundation: thread one diurnal multiplier `m` and a per-timestep
base air temperature `base(t)` into the existing `elemental pure feels_like_c`, add the
editable `base_*` / scenario-delta block to `coeffs_t` + `coeffs.nml` (loaded and validated),
and create `diurnal_mod` so the driver evaluates the grid at 4 times of day (TIME-01).

This plan owns the two foundational changes the rest of Phase 3 builds on: the
`feels_like_c` signature change (adds `m` as the 2nd argument, per D-01) and the `coeffs_t` /
namelist additions (D-03, D-04, D-05). It also adds the scenario deltas to config now so the
scenario plan (03-02) consumes them without re-touching the loader.

Purpose: TIME-01 — evaluate the grid at multiple times of day via a time-dependent diurnal
multiplier, with base(t) carrying absolute realism and m(t) carrying the urban-rural gap (D-02).
Output: extended `coeffs_t`/`coeffs.nml`, threaded `feels_like_c`, new `diurnal_mod`, a driver
timestep loop, and selector/regression tests. After this plan, `m = 1.0_wp` reproduces Phase-2
behavior exactly (regression locked by the unchanged Phase-2 test assertions).
</objective>
