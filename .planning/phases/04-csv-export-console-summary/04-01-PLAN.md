## Trust Boundaries

This is a self-contained local Fortran batch program. Input is a pre-validated local data file
(Phase 1). This plan produces library code (CSV writer + reductions) consumed in 04-02; it opens
no network and reads no untrusted/remote input.

| Boundary | Description |
|----------|-------------|
| in-memory grid/feels arrays → `results.csv` file | program data crosses into a written CSV file via formatted `write` |
| district `name` strings → CSV `name` field | static seed text (controlled) crosses into a delimited field |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | `name` column in write_results_csv | accept | District names are controlled static seed data (Phase 1 validated, contain no `,` or `"`), so delimiter/quote collision ("CSV injection") cannot occur on supported data. Quoting/escaping is explicitly deferred to v2 (D-05 keeps OUT-01 library-free). Low. |
| T-04-02 | Information disclosure / data integrity | formatted-write numeric fields | mitigate | Use width-free `F0.2`/`I0` descriptors: `F0.x` self-sizes (no field-overflow asterisk fill, A9) and `F` edit descriptors always emit `.` decimals regardless of locale (A10). Asserted by test_output. Low. |
| T-04-04 | Denial of service / integrity | hottest/coolest on empty occupied mask | mitigate | `count(g%cells%occupied) == 0` sentinel guard returns 0/0/0.0_wp instead of an out-of-bounds index from `maxloc`/`minloc` over an all-false mask. Low. |
| T-04-SC | Tampering (supply chain) | npm/pip/cargo/fpm installs | accept | No new dependencies added this plan; the only dev-dependency `test-drive` is pinned at tag `v0.6.0` (already in fpm.toml from Phase 1). No install step. |

No high-severity threats: no network, no auth, no remote/untrusted input, no runtime-derived file path.
